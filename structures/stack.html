Stack

  • pop() - Removes from top of stack
  • push() - Adds to top of stack
  • peak() / top() - Checks top of stack
  • count() - check size of stack
A stack is like a stack of plates (Last-In-First-Out or LIFO). The last plate you put on the stack is the first one you take off. It's commonly used in computer science for tasks where you want to keep track of things in reverse order, like navigating through function calls in a program. Think of it like a stack of trays at a cafeteriaâ€”the last tray you put on the stack is the first one you grab.

The space complexity of a stack can vary based on how it's implemented. In object-oriented programming (OOP), it might have a space complexity of O(n), but if you use an array with a fixed size, it's O(1). Because you only interact with the top item of the stack at any given time, the time complexity for push (adding an item) and pop (removing an item) operations is always O(1), regardless of the implementation method.
