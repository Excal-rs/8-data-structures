Binary Tree

  • Insert Node: Adds a new node with a specific key to the binary tree.
  • Delete Node: Removes a node with a specific key from the binary tree.
  • Search Node: Retrieves a node with a specific key from the binary tree.
  • Inorder Traversal: Visits all nodes of the binary tree in ascending order of their keys.
Binary Trees have a time complexity that varies for different operations. Accessing a specific node based on its key in a binary tree has an average and worst-case time complexity of O(log n), where n is the number of nodes in the tree. However, in the worst case, it can be O(n) for unbalanced trees. Inserting a new node and deleting a node from a binary tree also have a time complexity of O(log n) on average, but O(n) in the worst case for unbalanced trees.

The space complexity of a Binary Tree is O(n) in the worst case, where n is the number of nodes. This is because the tree structure is inherently recursive, and each node requires memory. In the best case, for a completely balanced tree, the space complexity is O(log n) due to the logarithmic height of the tree.
